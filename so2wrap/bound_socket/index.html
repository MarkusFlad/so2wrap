ยงBoundSocket: Thread-Safe, Rebindable TCP Listener Wrapper

Async-Wrapper for a [BoundSocket] that allows exclusive, sequential acquisition of the socket. Accessing the actual socket always requires acquiring a [ListeningSocket].

The key invariants of this design are:

  1. Exclusivity: Only one task can hold a [ListeningSocket] at any given time.
  2. Auto-Rebind: When the [ListeningSocket] is dropped, the underlying socket is atomically closed and a new one is immediately created and bound to the same address.
  3. Notification: Waiting tasks are notified and can immediately retry acquiring the socket via BoundSocket::listen().

This pattern is ideal for services that need to temporarily release the listening state but must guarantee continuous reservation of the bound TCP port while ensuring only one component can listen on it at a time.
